# Role

The roadmap prioritizes only already accepted tasks from `adm/pln/accepted/`. Idea capture and proposal assessment run through:

1. `adm/dev/protocols/workflow.md`
2. `adm/pln/inbox/_template.md`

# Planning Structure

1. `adm/pln/inbox/`: new proposals (`kind: Proposal`, `status: draft`)
2. `adm/pln/assessed/`: analyzed proposals (`kind: Proposal`, `status: assessed`)
3. `adm/pln/accepted/`: tasks (`kind: Task`, `status: none|open|wip|blocked`)
4. `adm/pln/rejected/`: rejected proposals or tasks (`status: done`, `tags: [rejected]`)
5. `adm/pln/realized/`: completed tasks (`status: done`, `tags: [realized]`)

Detailed transition rules and the mandatory proposal-assessment prompt are defined in `adm/dev/protocols/workflow.md`.
