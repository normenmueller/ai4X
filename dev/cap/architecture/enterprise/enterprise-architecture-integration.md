# Enterprise Architecture Integration (MUST)

## Purpose

Create enterprise-wide integration, interoperability, governance, and roadmap coherence across multiple architecture segments.

## Trigger

Use when architecture work must align principles, policies, roadmaps, or interoperability across multiple segments instead of one local domain or one solution.

## Rules (MUST)

- Distinguish enterprise-wide principles and policies from segment-local constraints and delivery choices.
- Make cross-segment integration and interoperability dependencies explicit before roadmap commitments are made.
- Use enterprise roadmaps to coordinate architectural change across multiple segments, not as a renamed project backlog.
- Keep governance of architectural change visible through explicit principles, policies, and coordination points.

## Fallback

- If segment scope, shared dependencies, or enterprise constraints are unclear, keep the output provisional and name the missing integration decisions.

## Minimal Output Contract

- enterprise architecture concern
- cross-segment dependencies
- governing principles or policies
- roadmap implication
