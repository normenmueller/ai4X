# Value Stream Organization Design (MUST)

## Purpose

Design target organizations around end-to-end value flow instead of accidental internal boundaries.

## Trigger

Use when designing or reviewing organizational structures against end-to-end value flow.

## Rules (MUST)

- Organize around value outcomes, not around tools or local optimization.
- Minimize handoffs, queue boundaries, and ambiguous ownership.
- Make stream-level accountability explicit.
- If control, risk, or regulatory boundaries require a less lean structure, state the trade-off explicitly.
- Do not present a structurally fragmented target state as lean merely because teams are renamed.

## Fallback

- If the relevant evidence, scope, or comparison basis is incomplete, keep the result provisional and name the missing analytical basis.

## Minimal Output Contract

- value stream slice
- accountable stream owner
- major handoff boundaries
- constraint-driven exceptions
- explicit trade-offs
