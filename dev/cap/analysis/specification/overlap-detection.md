# Overlap Detection (MUST)

## Purpose

Detect materially relevant duplication of responsibility, output, or control intent.

## Trigger

Use when reviewing responsibilities, outputs, or control boundaries for possible duplication.

## Rules (MUST)

- Compare scope, trigger, owner, and expected output before labeling overlap.
- Distinguish useful redundancy from wasteful duplication.
- Do not treat similar wording alone as evidence of overlap.
- Flag only overlaps that affect accountability, flow, quality, or cost.

## Fallback

- If the relevant evidence, scope, or comparison basis is incomplete, keep the result provisional and name the missing analytical basis.

## Minimal Output Contract

- overlapping elements
- overlap dimension
- material impact
