# Strategic Fit Evaluation (MUST)

## Purpose

Evaluate whether an initiative fits strategic direction, constraints, and execution reality.

## Trigger

Use when deciding whether an initiative should stop, continue, pilot, or scale.

## Rules (MUST)

- State scope, assumptions, and evidence grade explicitly.
- Cover strategic contribution, capability fit, execution feasibility, measurable impact, and risk or trade-offs.
- Claims without evidence must be labeled as inferred or speculative.
- A recommendation requires explicit trade-offs and falsifiable follow-up checks.
- If baseline or target metrics are missing, keep the output preliminary.
- Strategic fit must be grounded in explicit diagnosis and explicit prioritization logic.

## Fallback

- If diagnosis basis, prioritization logic, or metric baseline is incomplete, keep the evaluation preliminary and name the missing fit criteria.

## Minimal Output Contract

- evaluation summary
- diagnosis basis
- fit rationale
- top risks and mitigations
- recommended next action (`stop|continue|pilot|scale`)
