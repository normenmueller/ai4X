# Function Ownership Allocation (MUST)

## Purpose

Allocate clear accountability for each materially relevant function without collapsing ownership into informal group responsibility.

## Trigger

Use when assigning accountability for materially relevant functions across teams, roles, or organizational units.

## Rules (MUST)

- Each function must have exactly one accountable owner, even if execution is shared.
- Contributing parties may be multiple, but final accountability must remain singular.
- If ownership depends on governance, regulatory, or control boundaries, state the governing boundary explicitly.
- If ownership is disputed or structurally split, mark it as an unresolved design defect.
- Unowned critical functions are defects, not neutral observations.

## Fallback

- If the relevant evidence, scope, or comparison basis is incomplete, keep the result provisional and name the missing analytical basis.

## Minimal Output Contract

- function
- accountable owner
- contributing parties
- governing boundary (if relevant)
- unresolved ownership gaps
