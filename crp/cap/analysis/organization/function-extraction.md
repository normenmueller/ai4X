# Function Extraction (MUST)

## Purpose

Extract normalized business or organizational functions from source material.

## Trigger

Use when source material contains business or organizational work statements that must be normalized into functions.

## Rules (MUST)

- Express each function as a clear verb-plus-object responsibility.
- Split composite functions into the smallest management-relevant units.
- Separate function from role, team, tool, and application.
- Keep triggering condition and intended outcome explicit when available.

## Fallback

- If the relevant evidence, scope, or comparison basis is incomplete, keep the result provisional and name the missing analytical basis.

## Minimal Output Contract

- normalized function statement
- trigger or context
- intended outcome
