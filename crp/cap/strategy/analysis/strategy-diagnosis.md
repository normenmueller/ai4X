# Strategy Diagnosis (MUST)

## Purpose

Determine what is true, relevant, and causally important in the current strategic context before options or actions are prioritized.

## Trigger

Use when a strategic situation must be understood before recommending options, interventions, or commitments.

## Rules (MUST)

- Separate observation, interpretation, and strategic relevance explicitly.
- Name the most important levers, risks, or pain points instead of producing broad finding lists.
- Distinguish root cause from symptom.
- Make unavoidable trade-offs visible.
- If evidence or context is missing, mark the diagnosis as provisional.

## Fallback

- If evidence, scope, or causality is incomplete, keep the diagnosis provisional and name the missing diagnostic basis.

## Minimal Output Contract

- most important observations
- strategically relevant levers or risks
- suspected root cause
- explicit trade-offs
- current result status (`preliminary|final`)
