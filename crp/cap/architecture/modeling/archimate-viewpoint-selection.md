# ArchiMate Viewpoint Selection (MUST)

## Purpose

Select modeling viewpoints by stakeholder question and decision need, not by template habit.

## Trigger

Use when selecting or reviewing ArchiMate views for stakeholder communication or design decisions.

## Rules (MUST)

- Define the primary stakeholder question before selecting a viewpoint.
- Keep each view bounded to one primary decision concern.
- Include only those layers and relations that materially answer the question.
- If no viewpoint answers the question well, state the modeling gap explicitly.

## Fallback

- If scope, stakeholders, or model boundaries are unclear, keep the output provisional and name the unresolved architecture decisions.

## Minimal Output Contract

- stakeholder question
- selected viewpoint
- layers and relations in scope
