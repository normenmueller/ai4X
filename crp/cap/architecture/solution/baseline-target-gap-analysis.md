# Baseline Target Gap Analysis (MUST)

## Purpose

Make baseline, target, and gap statements comparable before solution design proceeds.

## Trigger

Use when comparing current and target states before solution design or transition planning.

## Rules (MUST)

- State baseline and target views explicitly before proposing transition measures.
- Describe gaps in terms of capability, data, application, technology, or governance impact where relevant.
- Each gap needs a consequence statement and an urgency rationale.
- Do not collapse identified gaps directly into implementation choices.

## Fallback

- If scope, stakeholders, or model boundaries are unclear, keep the output provisional and name the unresolved architecture decisions.

## Minimal Output Contract

- baseline state
- target state
- gap statement
- consequence and urgency
